# 购物中心 / 外卖插件 v17 说明

本版本保留“购物/外卖功能”独立开关，并把开关改成更轻量的标题栏按钮：

- 入口仍然保留在聊天工具栏的“购物中心”。
- 打开购物中心后，标题“购物中心”右侧有一个小按钮，只显示 `开` 或 `关`。
- 点击一次切换开启，再点击一次切换关闭。
- 切换后顶部提示只显示：`购物中心已打开` 或 `购物中心已关闭`。
- 不再显示大块说明卡片，也不再显示旧版那种占空间的说明。
- 关闭时：普通触发 AI 不会注入购物/外卖提示词，不解析 `[[NUOMI_COMMERCE: ...]]`，也不拦截原版转账动作，普通聊天尽量保持原版。
- 开启时：购物/外卖提示词才会加入，char 可以主动买东西、主动点外卖，也可以响应外卖代付请求并弹出支付/拒绝卡片。
- 开关保存在浏览器本地：`nuomi_commerce_feature_enabled_v5`。
- 数据导出/导入仍然在购物中心内部，包含购物/外卖商品、分类、AI 补货 API 设置、API Key、预设以及这个开关状态。

v16 已修复本地开发时 Vite 误扫描项目里其他 HTML 文件导致的红字提示；v17 继续保留这个设置。

v17 新增：购物中心内部顶部提示会自动隐藏。像 `购物中心已打开`、`购物中心已关闭`、`已保存到分类`、`预设已保存` 这类提示会显示约 3 秒，然后淡出消失，不会一直堆在顶部。

测试方式：

```bash
npm install
npm run dev
```

如果同一个文件夹之前已经安装过依赖，通常只需要：

```bash
npm run dev
```
