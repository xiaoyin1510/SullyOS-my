01

新增 · 「心象」 NEW

让你看见模型的思考。

开启后，角色每次回复，气泡顶部会浮出一张「心象」卡片——里面是 AI 模型在准备这条回复时自己原生输出的思考链（OpenAI 协议里的 reasoning_content 或 <think> 标签内容）。我们用 prompt 尽量把它染成角色第一人称的内心独白。

开启位置

  1. 进入任意角色的聊天界面。
  2. 点击输入框左侧的＋号，弹出操作面板。
  3. 滑到第二页（麦当劳 / HTML 模式所在的那页）。
  4. 点击展示思考按钮——会直接打开「心象」设置弹窗，里面有总开关、卡片风格、自定义配色、追加提示词。

卡片风格

  • 心象（默认）：暗紫 × 暖金，二次元游戏卡牌质感。
  • 心声：羊皮纸暖色，私密日记的氛围。
  • 极简：纯白单色，OOC 调试视图。
  • 自定：背景 / 点缀 / 正文三色调教，背景支持 CSS 渐变。

追加自定义提示词

原生提示词（让模型用角色第一人称、中文意识流思考）保持不变；你写的内容追加在最后作为「用户对内心独白的额外要求」。比如让 ta 思考时偶尔切到日语、多写感官细节、用昵称叫你等等。

关于「心象」的本质

这是模型自己原生的思考链，不是我们额外让模型为角色生成的"内心戏"。它的本质决定了它不会像角色台词那样鲜活——更像看一个演员在化妆间的喃喃自语，而不是舞台上的台词。这个功能本来就是给"喜欢看大模型思维链"的用户准备的彩蛋，不一定适合每个人，觉得跳戏直接关掉就好。

另外：思维链不进入上下文，也不会成为角色真实回复的一部分——所以下一轮，角色不会记得自己上一轮在想什么，ta 只会基于真正发出去的回复（+ 对话历史）往下走。

没看到「心象」卡片？

最常见的原因是当前模型不带思考链能力——请问你 API 提供者哪些模型支持 thinking，或自己查找。次常见的原因是这一轮模型没思考（短回复 / 模型自己判断不需要），属于正常现象，下一轮可能就有。如果是代理拒绝转发 thinking 字段，跟 API 提供方确认对应模型是否启用了 extended thinking。

02

修复 · 「约会（见面模式）」上下文与重发体验 FIX

不再 400，按一下就重发。

之前在「约会（见面模式）」里偶尔会遇到 prompt is too long（上下文超 200K tokens）的 400 报错，明明聊天界面里看 token 才四万多。原因是「约会（见面模式）」自己写的历史构建器没对齐聊天：

  • 开了向量记忆不按高水位线裁剪，旧消息全文回放
  • 表情包 / HTML 卡片 / 转发等没按类型脱水，原始 content 直接塞进 prompt

现在两端共用同一份精细脱水逻辑，token 占用回到聊天侧的同一量级。

另外修复了「API 失败后必须重新打字才能重试」的体验问题。现在如果上一轮发出去 API 报错，输入框为空时按发送键，按钮会变成 RETRY，直接拿上一条已存的内容重新触发 LLM，不会重复落库。

这就是 5 月 10 日的小更新

如果「心象」让你跳戏，关掉就好；如果让你着迷，欢迎调成自己的味道。
